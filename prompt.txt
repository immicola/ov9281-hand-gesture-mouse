We are optimizing latency for a Thor robot arm controlled via gesture recognition.
## Current setup
- Raspberry Pi 4 (OV9281 camera) runs `robo_arm_v2.py` — MediaPipe gesture → UART → Pico → USB → WSL
- WSL (Windows) runs `thor_bridge.py` — reads serial from Pico, publishes via WebSocket to rosbridge
- Thor-ROS runs in WSL, hardware interface communicates with the real robot via usbipd (USB over IP from Windows)
## What has been optimized already (on Pi 4 side)
- Pico bridge: removed `time.sleep(0.005)` in code.py
- robo_arm_v2.py: CMD_THROTTLE 0.5 → 0.15, X_THROTTLE 0.1 → 0.03, now processing every frame
- thor_bridge.py: socket timeout 0.2 → 0.01, PUBLISH_THROTTLE 0.04 → 0.01, serial timeout 0.1 → 0.01
- thor_bridge.py: added event-driven publishing (changed_event) — data changes trigger immediate publish instead of polling
## Current problem
Signals reach thor_bridge.py fast (confirmed), but the arm moves 1-3 seconds late. Commands "accumulate" — the arm jerks after a delay instead of moving immediately. The bottleneck is in the Thor-ROS → ros2_control → hardware interface → usbipd → robot path inside WSL.
